## 一、首页 UI 重设计

### 当前问题

- 2 列网格无固定高度，项目多时无限拉长
- 卡片 `p-5` 偏大，信息密度低
- 无收藏功能
- 无列表视图
- 注册表单占一整行
- 两侧大片空白无内容

### 目标布局

```
┌──────────────────────────────────────────────┐
│ 📁 Doc77                     🌙 主题  ⚙ 设置  │  ← Header
├──────────────────────────────────────────────┤
│ 👋 下午好         [🔍 发现项目] [+ 注册项目]  │  ← 问候 + 快捷操作
├────────────────────┬─────────────────────────┤
│ 🕐 最近浏览         │ 📊 统计                 │
├────────────────────┴─────────────────────────┤
│ ⭐ 收藏项目 (N)                               │  ← 无收藏时整区隐藏
│ [compact cards · 2-col grid]                  │
├──────────────────────────────────────────────┤
│ 📂 全部项目 (N)         [🔍 搜索] [📋≡ 切换] │
│ [cards · max-h-[50vh] · overflow-y-auto]     │
└──────────────────────────────────────────────┘
```

## 三、新增 API

### `/api/stats`

```json
{
  "projects": 3,
  "lastActive": "2 分钟前"
}
```

从 `projects` 表 COUNT + `projects.last_opened` MAX 得出。不统计文件数（缓存值不准确，不如不显示）。

### `/api/discover?path=~/projects&depth=2`

```json
[
  { "name": "doc77", "path": "/home/user/work/doc77", "hasReadme": true },
  { "name": "my-notes", "path": "/home/user/work/my-notes", "hasReadme": false }
]
```

## 四、移动端同步

| 功能 | 桌面端 | 移动端 |
|---|---|---|
| 收藏项目 | ✅ localStorage `doc77-favorites` | ✅ 共享同 key，项目卡片长按弹出收藏/取消 |
| 问候语 | ✅ | ✅ topbar 下方 |
| 统计 | ✅ 项目数 + 最后活跃 | ✅ 简化（仅项目数） |
| 注册项目 | ✅ 弹窗 | ❌ 不做（桌面端专属操作） |
| 发现项目 | ✅ 弹窗内 Tab | ❌ 不做 |

### 数据格式

**收藏**：`doc77-favorites = [projectId, ...]`（数组，按收藏时间排序）

**最近浏览**：复用现有 `doc77_recent_files`，渲染时过滤掉已删除项目的残留数据。

## 五、边界情况 & 设计决策

| 场景 | 处理方式 |
|---|---|
| 统计「文件数」准确性问题 | 不显示文件数，只显示项目数 + 最后活跃时间 |
| 项目已删除但收藏/最近仍引用 | 渲染时过滤：`projects.find(p => p.id === favId)`，不存在则跳过 |
| 注册弹窗「发现」Tab 扫描目录 | 默认 `~`，用户可手动修改路径 |
| 移动端收藏操作 | 长按项目卡片弹出「⭐ 收藏 / ✕ 取消收藏」 |
| 注册弹窗关闭时未保存 | 关闭弹窗不自动保存，用户必须点击 [注册] 或 [批量注册] |
| 发现扫描超时 | 限制深度 + 跳过 `node_modules`/`.git` 等大目录，超时 10s 返回已有结果 |

### CLI

```bash
doc77 discover                    # 默认扫描 ~/，深度 2 层
doc77 discover --path ~/work      # 指定扫描目录
doc77 discover --depth 3          # 指定深度
```
