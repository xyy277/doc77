## 一、方案选择

**选择方案 B：适度重构 + CSS 统一。**

桌面端 CSS 从 Tailwind CDN 迁移到自定义 CSS 变量体系（与移动端一致的 Design Token），JS 拆分为多个职责清晰的模块。

| 对比维度 | 方案 A（增量叠加） | 方案 B（适度重构）| 方案 C（SPA 重写） |
|---|---|---|---|
| 改动量 | 最小 | 中等 | 大 |
| JS 文件可维护性 | 单文件 600+ 行 | 多文件，各 ~100-200 行 | 组件化 |
| CSS 一致性 | 桌面/移动分裂 | **统一 Design Token** | 统一 |
| 预估时间 | 2-3 天 | **3-4 天** | 7-10 天 |
| 风险 | 低 | 中（需 CSS 回归测试）| 高 |

## 三、数据模型变更

### 3.1 SQLite Migration

```sql
-- 收藏表
CREATE TABLE IF NOT EXISTS favorites (
    project_id INTEGER NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (project_id),
    FOREIGN KEY (project_id) REFERENCES projects(id) ON DELETE CASCADE
);

-- 最近浏览文件表（替代 localStorage doc77_recent_files）
CREATE TABLE IF NOT EXISTS recent_files (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    project_id INTEGER NOT NULL,
    file_name TEXT NOT NULL,
    file_path TEXT NOT NULL,
    viewed_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (project_id) REFERENCES projects(id) ON DELETE CASCADE
);

-- 索引
CREATE INDEX IF NOT EXISTS idx_favorites_created ON favorites(created_at);
CREATE INDEX IF NOT EXISTS idx_recent_files_viewed ON recent_files(viewed_at DESC);
CREATE INDEX IF NOT EXISTS idx_projects_last_opened ON projects(last_opened);
```

### 3.2 设计决策

- **独立 `favorites` 表而非 `projects.favorite` 列**：收藏是关系，独立表语义更清晰，`ON DELETE CASCADE` 自动清理
- **`recent_files` 替代 localStorage**：数据不丢失，`ON DELETE CASCADE` 保证删除项目后自动清理残留，无需前端过滤
- **`last_opened` 已有索引**：项目级「最近浏览」直接用，无需新增存储

## 四、API 设计

### 4.1 新增端点

| 方法 | 路径 | 功能 | 请求 | 响应 |
|---|---|---|---|---|
| `PUT` | `/api/projects/:id/favorite` | 切换收藏 | — | `{ id, favorited: bool }` |
| `GET` | `/api/stats` | 统计 | — | `{ projects, lastActive, favoriteCount }` |
| `GET` | `/api/discover` | 扫描候选项目 | `?path=~&depth=2` | `[{ name, path, hasReadme, mdCount }]` |
| `POST` | `/api/recent-files` | 记录文件浏览 | `{ projectId, fileName, filePath }` | `201` |
| `GET` | `/api/recent-files` | 最近文件 | `?limit=5` | `[{ fileName, filePath, projectId, projectName, viewedAt }]` |

### 4.2 端点详细设计

#### `GET /api/discover?path=~&depth=2`

**跳过规则（跨平台）：**

| 平台 | 跳过目录 |
|---|---|
| Linux / macOS | 以 `.` 开头（`.config`, `.local`, `.cache` 等） |
| Windows | `AppData`, `Application Data`, `Cookies`, `NetHood`, `PrintHood`, `Recent`, `SendTo`, `Start Menu`, `Templates`, `Local Settings` |
| 全平台 | `node_modules`, `__pycache__`, `.git`（内容跳过，但 `.git` 存在 = 项目信号） |

**判定逻辑**：候选目录需同时满足：
1. 包含 `.git` 子目录
2. 包含 ≥ 1 个 `*.md` 文件（仅检查当前目录层级，不递归）

**超时**：10 秒后返回已有结果，不报错。

**路径安全**：拒绝系统目录（`/sys`, `/proc`, `/dev`, `C:\Windows`, `C:\Program Files` 等），黑名单硬编码。

#### `GET /api/stats`

```json
{
  "projects": 3,
  "lastActive": "2026-07-10T08:30:00Z",
  "favoriteCount": 2
}
```

- `projects`: `SELECT COUNT(*) FROM projects`
- `lastActive`: `SELECT MAX(last_opened) FROM projects`（ISO 8601，前端渲染为相对时间）
- `favoriteCount`: `SELECT COUNT(*) FROM favorites`

#### `GET /api/recent-files?limit=5`

```json
[
  {
    "fileName": "API设计.md",
    "filePath": "技术文档/API设计.md",
    "projectId": 1,
    "projectName": "doc77",
    "viewedAt": "2026-07-10T08:30:00Z"
  }
]
```

**自动过滤**：`JOIN projects` 确保已删除项目的记录不返回（`ON DELETE CASCADE` + JOIN 天然过滤）。

### 5.2 Tailwind → 自定义 CSS 映射

当前 Tailwind 用法集中在工具类，迁移方式：

| Tailwind | 自定义 CSS |
|---|---|
| `bg-white dark:bg-slate-800` | `background: var(--bg-card)` |
| `border border-slate-200 dark:border-slate-700` | `border: 1px solid var(--border-light)` |
| `rounded-xl` | `border-radius: 12px` |
| `p-5` | `padding: 20px` |
| `text-sm`, `text-xs` | `font-size: 14px` / `12px` |
| `shadow-sm` | `box-shadow: 0 1px 2px rgba(0,0,0,0.05)` |
| `hover:shadow-lg` | `.card:hover { box-shadow: ... }` |
| `transition-all` | `transition: all .2s` |
| `truncate` | `overflow:hidden;text-overflow:ellipsis;white-space:nowrap` |
| `flex gap-3 items-center` | `display:flex;gap:12px;align-items:center` |
| `font-semibold` | `font-weight: 600` |
| `max-w-5xl mx-auto` | `max-width: var(--max-content-width); margin: 0 auto` |

**原则**：
- 公共模式提取为 class（`.card`, `.btn`, `.input` 等）
- 一次性布局用内联 `style` 或特定 ID 选择器
- 不再引入 Tailwind CDN

## 九、边界情况与设计决策

| 场景 | 处理方式 |
|---|---|
| 项目已删除，`recent_files` / `favorites` 残留 | `ON DELETE CASCADE` 自动清理，JOIN 查询天然过滤 |
| 发现扫描超大目录 | 深度限制（默认 2）+ 隐藏/系统目录跳过 + 10s 超时 |
| `discover` 返回已注册项目 | 前端标记「已注册」并禁用勾选，无需后端去重 |
| 注册弹窗关闭 | 不自动保存，用户必须点击 [注册] 或 [批量注册] |
| 移动端长按 → 改为显式 ⭐ | 提高可发现性，与桌面端一致的星标交互 |
| Windows 路径处理 | 复用现有 `resolvePath`（支持 `~`、`C:\`、WSL 互转） |
| `last_opened` 为 NULL | 统计面板「最近活跃」显示「暂无记录」 |
| `recent_files` 表持续增长 | 保留最近 50 条，`POST` 时触发 `DELETE` 清理旧记录 |
